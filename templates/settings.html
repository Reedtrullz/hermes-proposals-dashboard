Worker setup

Proposals are stored locally in SQLite. Saving a real proposal writes its id to the existing trigger file, but this dashboard does not itself execute a worker or call a paid LLM provider.

1.

Use Try demo on the Proposals page to explore review and decision controls without executing anything.

2.

Configure an external Hermes worker or CLI worker to consume $HERMES_HOME/proposals_trigger and, when present, proposals_trigger_executor.

3.

When you approve a real proposal, the dashboard writes APPROVED:<id> through the existing integration.
